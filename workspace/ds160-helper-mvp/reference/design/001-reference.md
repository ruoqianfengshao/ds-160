# DS-160 Helper MVP 产品设计要求

## 产品定位
美国签证DS-160申请表辅助填写工具，核心价值是将官方表单填写体验从"痛苦且容易出错"变为"引导式、可保存、无感同步"。

## MVP范围（Phase 0.5 - 2周目标）

### 核心功能
1. **表单引导填写**（仅Web端）
   - 12个步骤的分步表单
   - 每个字段附带中文白话解释
   - 高风险字段标红警示
   - 实时一致性校验

2. **草稿管理**
   - 本地存储（localStorage）
   - 自动保存
   - 可随时恢复

3. **免费版权益**
   - 1份草稿
   - 完整的12步表单填写
   - 3次"模拟同步"体验（实际不连接DS-160官网）

### 技术方案

#### 前端架构
- **框架**：Nuxt 3（支持SSR/CSR混合）
- **构建工具**：Vite 8
- **状态管理**：Pinia
- **样式**：Tailwind CSS v4
- **类型**：TypeScript

#### 页面划分
**SSR页面**（营销）：
- 首页（/）
- 功能介绍（/features）
- 定价页（/pricing）

**CSR页面**（应用）：
- 用户登录/注册（/auth/login, /auth/register）
- 应用面板（/app/dashboard）
- 表单填写（/app/form/step-[1-12]）
- 草稿管理（/app/drafts）

#### 数据结构（简化版）
```typescript
interface Draft {
  id: string
  createdAt: Date
  updatedAt: Date
  currentStep: number
  data: {
    personalInfo: {...}
    travelInfo: {...}
    previousTravel: {...}
    // ... 其余步骤
  }
  syncCount: number // 已使用的同步次数
}
```

### 12个表单步骤（简化版）

1. **个人信息**：姓名、性别、出生日期、出生地
2. **护照信息**：护照号、签发日期、有效期
3. **联系方式**：地址、电话、邮箱
4. **旅行信息**：旅行目的、预计停留时间、入境日期
5. **旅行伙伴**：是否同行、同行人信息
6. **美国联系人**：在美联系人或酒店信息
7. **家庭信息**：父母信息、配偶信息
8. **工作/教育**：当前职业、教育背景
9. **安全问题（1）**：犯罪记录、违法行为
10. **安全问题（2）**：恐怖主义、毒品相关
11. **以往旅行**：是否去过美国、是否被拒签
12. **照片上传与确认**：上传照片、最终确认

### 高风险字段提示示例

以下字段标红并显示警告：
- 犯罪记录（回答错误可能导致拒签）
- 拒签历史（必须如实回答）
- 社交媒体账号（需要提供）
- 以往赴美记录（需一致）

### UI/UX要求

#### 表单填写页面
- 左侧：步骤导航（显示12个步骤，当前高亮）
- 中间：当前步骤的表单字段（每步最多8个字段）
- 右侧：字段说明和填写建议
- 底部：[上一步] [保存草稿] [下一步]

#### 草稿保存提示
- 每次字段变更后3秒自动保存
- 保存成功显示"✓ 已保存"
- 保存失败显示"⚠️ 保存失败，请检查网络"

#### 同步体验（模拟）
- 点击"同步到DS-160官网"按钮
- 显示进度条：检查数据 → 连接官网 → 填充表单
- **实际操作**：仅计数+1，不真的连接官网
- 完成后显示"✓ 同步成功（剩余X次）"
